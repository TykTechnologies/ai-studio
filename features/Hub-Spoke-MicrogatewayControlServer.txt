│ │ Key Architectural Decisions                                                                                                                                                                          │ │
│ │                                                                                                                                                                                                      │ │
│ │ 1. Namespace Model                                                                                                                                                                                   │ │
│ │                                                                                                                                                                                                      │ │
│ │ - Empty string ("") = global namespace (fully backward compatible)                                                                                                                                   │ │
│ │ - Non-empty string = specific namespace for multi-tenancy                                                                                                                                            │ │
│ │ - Visibility Rules:                                                                                                                                                                                  │ │
│ │   - Global objects (namespace="") visible to all edges                                                                                                                                               │ │
│ │   - Namespaced objects only visible to edges with matching namespace                                                                                                                                 │ │
│ │   - Edges with empty namespace only see global objects                                                                                                                                               │ │
│ │                                                                                                                                                                                                      │ │
│ │ 2. Database Integration with GORM                                                                                                                                                                    │ │
│ │                                                                                                                                                                                                      │ │
│ │ - All new namespace fields have gorm:"default:''" for backward compatibility                                                                                                                         │ │
│ │ - GORM AutoMigrate will safely add columns without data loss                                                                                                                                         │ │
│ │ - Existing records automatically get empty string (global) namespace                                                                                                                                 │ │
│ │ - New composite indexes for efficient namespace filtering                                                                                                                                            │ │
│ │                                                                                                                                                                                                      │ │
│ │ 3. API Compatibility                                                                                                                                                                                 │ │
│ │                                                                                                                                                                                                      │ │
│ │ - All existing APIs continue to work unchanged                                                                                                                                                       │ │
│ │ - New namespace parameter is optional on all endpoints                                                                                                                                               │ │
│ │ - No breaking changes to existing API clients                                                                                                                                                        │ │
│ │ - Default behavior remains global scope when no namespace specified                                                                                                                                  │ │
│ │                                                                                                                                                                                                      │ │
│ │ 4. gRPC Control Server                                                                                                                                                                               │ │
│ │                                                                                                                                                                                                      │ │
│ │ - Runs on separate port from HTTP API (default 9090)                                                                                                                                                 │ │
│ │ - Only starts when GATEWAY_MODE=control is configured                                                                                                                                                │ │
│ │ - Supports TLS and token-based authentication                                                                                                                                                        │ │
│ │ - Graceful handling of edge disconnections and reconnections                                                                                                                                         │ │
│ │                                                                                                                                                                                                      │ │
│ │ Success Criteria                                                                                                                                                                                     │ │
│ │                                                                                                                                                                                                      │ │
│ │ 1. ✅ AI Studio can run in control mode with gRPC server active                                                                                                                                       │ │
│ │ 2. ✅ Microgateway edges can successfully connect and register with AI Studio                                                                                                                         │ │
│ │ 3. ✅ Configuration synchronization works bidirectionally between control and edges                                                                                                                   │ │
│ │ 4. ✅ Namespace-based filtering correctly isolates tenant configurations                                                                                                                              │ │
│ │ 5. ✅ Edge management APIs provide full visibility and control                                                                                                                                        │ │
│ │ 6. ✅ All existing AI Studio functionality remains completely unchanged                                                                                                                               │ │
│ │ 7. ✅ GORM AutoMigrate successfully adds new fields and models                                                                                                                                        │ │
